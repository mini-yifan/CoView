状态定义：
- `已支持`：能力可直接使用，有自动化验证。
- `部分支持`：能力可用但有已知限制，或依赖降级路径。
- `不支持`：当前版本明确不支持。

## 说明

1. Windows `open_app_launcher` 通过按 `Win` 键打开开始菜单；当前不扫描安装应用，返回空应用列表。  
2. Windows `launch_app` 当前会优先激活已运行实例，只有未找到运行实例或激活失败时才回退到直接启动路径。  
3. Windows `activate_app` 当前已支持优先按 `hwnd`、回退按 `pid` 的 best-effort 前台恢复，包含最小化恢复与线程附着兜底；受系统前台锁限制时仍可能失败，因此标记为部分支持。  
4. Windows `read_current_document` 为 copy-based V1：IDE/代码编辑器要求传 `screen_index` 和 `position`，`follow_view` 暂未实现。  
5. Windows `get_active_document_path` 当前支持 File Explorer、Word、Excel、PowerPoint、WPS；对未保存文档或非文件系统位置返回空字符串。  
6. Windows `move_to_trash` 当前通过 Shell API 将文件或目录移入系统回收站，不做永久删除。  
7. 新增或调整平台能力时，需同步更新本矩阵中的自动化测试和手动回归编号。
