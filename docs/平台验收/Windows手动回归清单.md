# Windows 手动回归清单

## 执行前检查

1. 机器为 Windows 10/11，应用可正常启动。
2. 已安装依赖，能运行自动化测试入口。
3. 若涉及系统控制能力，确认已授予必要权限。

## GUI 基础行为

1. `WIN-GUI-01` 悬浮球与输入面板  
步骤：启动应用，展开/收起面板，提交一次任务，再点击停止。  
预期：悬浮球、面板状态切换正常；任务可启动、可停止。

2. `WIN-GUI-02` 透明模式切换  
步骤：执行会触发自动化的任务，观察执行前后悬浮窗口。  
预期：执行前进入透明穿透；执行后恢复可交互。

3. `WIN-GUI-03` 日志窗口  
步骤：打开日志窗口，执行一条任务，观察日志滚动。  
预期：日志持续刷新，无明显卡死；窗口可关闭。

## 平台能力行为

1. `WIN-PLT-01` 截图链路  
步骤：执行需要多轮观察的任务。  
预期：任务能持续推进；无“截图失败”中断。

2. `WIN-PLT-02` 前台应用识别  
步骤：切换多个前台窗口后执行任务。  
预期：可识别前台窗口变化，行为与当前窗口上下文基本一致。

3. `WIN-PLT-03` 打开应用  
步骤：分别请求打开“命中应用”和“模糊应用名”。  
预期：命中应用可拉起；模糊名返回候选提示。

4. `WIN-PLT-04` 打开应用启动器  
步骤：触发 `open_app_launcher`。  
预期：当前版本返回“不支持”或等价失败信息（这是已知边界）。

5. `WIN-PLT-05` 打开文件/目录  
步骤：分别传目录路径、文件路径调用 `open_in_finder`。  
预期：目录可在资源管理器打开；文件可被定位。

6. `WIN-PLT-06` 打开 URL/搜索词  
步骤：分别传 `url` 和 `query` 调用 `open_in_browser`。  
预期：默认浏览器打开目标页面；`query` 被正确转为搜索 URL。

## 记录格式

每项按 `通过/失败/受限` 记录，并附：
- 失败定位点：`截图` / `窗口` / `权限` / `工具执行`
- 复现步骤编号
- 机器环境（Windows 版本、是否管理员、是否多屏）
