# macOS 手动回归清单

## 执行前检查

1. 机器为 macOS，应用可正常启动。
2. 已安装依赖，能运行自动化测试入口。
3. 系统权限已授权：辅助功能、屏幕录制（如未授权，先记录失败原因后再授权重试）。

## GUI 基础行为

1. `MAC-GUI-01` 悬浮球与输入面板  
步骤：启动应用，展开/收起面板，提交一次任务，再点击停止。  
预期：悬浮球、面板状态切换正常；任务可启动、可停止。

2. `MAC-GUI-02` 透明模式切换  
步骤：执行会触发自动化的任务，观察执行前后悬浮窗口。  
预期：执行前进入透明穿透；执行后恢复可交互。

3. `MAC-GUI-03` 日志窗口  
步骤：打开日志窗口，执行一条任务，观察日志滚动。  
预期：日志持续刷新，无明显卡死；窗口可关闭。

## 平台能力行为

1. `MAC-PLT-01` 截图链路  
步骤：执行需要多轮观察的任务。  
预期：任务能持续推进；无“截图失败”中断。

2. `MAC-PLT-02` 前台应用识别  
步骤：先切到浏览器/访达，再发任务让系统读取当前前台应用上下文。  
预期：行为与当前前台应用一致，未出现明显误判。

3. `MAC-PLT-03` 打开应用（含 fallback）  
步骤：分别请求打开“命中应用”和“模糊应用名”。  
预期：命中应用可拉起；模糊名给出候选或引导 `open_app_launcher`。

4. `MAC-PLT-04` 打开启动台  
步骤：触发 `open_app_launcher`。  
预期：启动台被拉起，可返回应用列表。

5. `MAC-PLT-05` 打开文件/目录  
步骤：分别传目录路径、文件路径调用 `open_in_finder`。  
预期：目录直接打开；文件在访达中被定位/选中。

6. `MAC-PLT-06` 打开 URL/搜索词  
步骤：分别传 `url` 和 `query` 调用 `open_in_browser`。  
预期：默认浏览器打开目标页面；`query` 被正确转为搜索 URL。

## 记录格式

每项按 `通过/失败/受限` 记录，并附：
- 失败定位点：`截图` / `窗口` / `权限` / `工具执行`
- 复现步骤编号
- 机器环境（macOS 版本、芯片、是否多屏）
